Grunt and the Command Line

Why use it?

  1. Because it's what all the kids are doing
  2. If you have errors, you can see what is going wrong and fix it

Command Line (Terminal on Mac):

  1. Open it
  2. Traverse to the file you are working on
    1. type "cd **folder name**" (cd means change directory)
    2. ex. if you are working on your desktop you will type:
      1. cd desktop
      2. cd **project folder**

There are 3 commads you need to know to run the build system

  1. npm install (run this the first time you start the project)
  2. grunt serve (run this to work on the files)
  3. grunt build (run this when you are ready to push your files)

Debugging Grunt

  1. When you are working on the files or ready to push and you encounter an error type the command you are running and add --verbose
    1. grunt serve --verbose
    2. This will tell you every task that has ran and stop showing you an error ex.
    3. This example show that on build, copy > dist was missing. Going into the gruntfile we can see this has been commented out. If you have less/css errors, build errors, or anything go side ways. The command line will tell you what is going on.
